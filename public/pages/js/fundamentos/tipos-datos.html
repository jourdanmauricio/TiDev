Tipos de datos y el operador typeof

Todo dato (valor) de una variable o constante pertenece a un tipo. En JavaScript existen ocho tipos de datos básicos.

  • Number
  • BigInt
  • String
  • Boolean
  • Null
  • Undefined
  • Object
  • Symbol

JavaScript es un lenguage dinámicamente tipado. Esto quiere decir que podemos almacenar cualquier tipo dentro de una variable y luego cambiar el valor por otro tipo de dato.

        
          // no genera error
          let message = "hola";
          message = 123456;

Tipo Number

El tipo de datos number representa tanto números enteros como de punto flotante (decimales) y podemos aplicar las operaciones numéricas: suma +, resta -, multiplicación *, división /, etc.

        
          let n = 213;
          n = 12.654;

Dentro de los valores númericos existen tres denominados "especiales": Infinity, -Infinity, NaN.

  • Infinity representa el Infinito matemático ∞. Es un valor especial que es mayor que cualquier número.

                
              alert( 1 / 0 ); // Infinity
              alert( Infinity ); // Infinity

  • NaN representa un error de cálculo. Es el resultado de una operación matemática incorrecta o indefinida, por ejemplo:

                
              alert( "no es un número" / 2 ); // NaN, tal división es errónea
              // Cualquier otra operación sobre NaN devuelve NaN:
              alert( NaN + 1 ); // NaN
              alert( 3 * NaN ); // NaN
              alert( "not a number" / 2 - 1 ); // NaN
JavaScript no arroja errores al dividir por cero, tratar las cadenas no numéricas como números, etc. En el peor de los casos, obtendremos NaN como resultado.

Cantidad de decimales

El método toFixed() retorna una representación de cadena con la cantidad de dígitos decimales indicados. 

        
          let numero = 12345.6789;
          numObj.toFixed();    // '12346'
          numObj.toFixed(1);   // '12345.7'
          numObj.toFixed(6);   // '12345.678900'

Parsear a entero

El método parseInt() analiza un argumento de cadena y devuelve un entero de la raíz o base especificada.

Este método nos ayudará cuando trabajamos con datos provenientes desde formularios HTML (cadenas) y queremos tratarlos como enteros. Si tenemos que realizar operaciones artiméticas con decimales ocupamos parseFloat(). 

        
          let numero = 12345.6789;
          console.log(parseInt(numero)); // 12345

          let numero = "1";
          let numero2 = 2;
          console.log(numero + numero2); // "12"
          console.log(parseInt(numero) + numero2); // 3

Tipo BigInt

El tipo number no puede representar valores enteros mayores que (253-1) o 9007199254740991, o menor que -(253-1) para negativos.

Un valor BigInt se crea agregando n al final de un entero:

        
          const bigInt = 1234567890123456789012345678901234567890n;

Raramente se utilizan números de tipo BigInt, generalmente se utilizan para criptografía o marcas de tiempo de precisión de microsegundos.

String

Un string en JavaScript es una cadena de caracteres y debe colocarse entre comillas. Existen 3 tipos de comillas.

  • Comillas dobles: "Hola".
  • Comillas simples: 'Hola'.
  • Template strings (comillas invertidas): `Hola`.

Las comillas dobles y simples son comillas “sencillas” (es decir, funcionan igual). No hay diferencia entre ellas.

Los comillas invertidas son comillas de funcionalidad extendida. Nos permiten incrustar variables y expresiones en una cadena de caracteres encerrándolas en ${...}.

Las comillas simples y las dobles no permiten saltos de línea dentro de la cadena (debemos escaparlos). Para crear una cadena con varias líneas ocupamos las comillas invertidas.

        
          let name = "John";

          // incrustar una variable
          alert( `Hola, ${name}!` ); // Hola, John!
          // Es equivalente a:
          alert( "Hola, " + name + "!" );

          // incrustar una expresión
          alert( `El resultado es ${1 + 2}` ); // el resultado es 3

          // Cadena de varias líneas
          let cadena = "Línea 1           // Genera un error
            Línea 2
            Línea 3"; 

          let cadena2 = `Línea 1        // Respeta la identación, espacios, saltos de línea
            Línea 2
            Línea 3`;

La expresión dentro de ${...} se evalúa y el resultado pasa a formar parte de la cadena. Se debe considerar que sólo se puede hacer con los backticks. ¡Las otras comillas no tienen esta capacidad de incrustación!

Longitud de la cadena

Podemos obtener la logintud de la cadena a travaés de la propiedad length.

        
          let banda = "Catupecu Machu";
          console.log(`Cantidad de caracteres: ${banda.length}`); // Cantidad de caracteres: 14

Acceder a un caracter de la cadena

Existen dos formas de accedera un caracter: charAt() y [].

        
          let banda = "Catupecu Machu";
          console.log(banda.charAt(2)); // Mostrará "t" 
          console.log(banda[2]); // Mostrará "t"

Comparar cadenas

JavaScript es sensible a mayúsculas y minúsculas, por lo que para comparar cadenas podemos utilizar los métodos toUpperCase o toLowerCase para comparar todos los caracteres en mayúsculas o minúsculas. 

        
          let name = 'Mauro'
          let name2 = 'MaurO'
          
          if (name.toUpperCase() === name2.toUpperCase()) { // true
            console.log('Son iguales');
          } else 
            console.log('Distintas');
          }

Escapando Caracteres

Cuando quiera incluir un caracter como una tabulación, comillas, salto de línea, etc se debe anteponer el caracter \.

        
          console.log ("Aprendiendo \"JavaScript\"");

Búsqueda en cadenas

Los métodos de búsqueda retornan true o false según se cumplan las condiciones.

El método includes() determina si una cadena de texto se encuentra dentro de otra cadena de texto.

El método startsWith() indica si una cadena de texto comienza con los caracteres de una cadena de texto concreta.

El método endsWith() determina si una cadena de texto termina con los caracteres de una cadena indicada. 

        
          let banda = "Catupecu Machu";
          console.log(banda.includes("pe")); // true 
          console.log(banda.startsWith("Ca")); // true 
          console.log(banda.endsWith("hu")); // true 
          console.log(banda.includes("m")); // false

El método indexOf() y search() retornan el índice de la primera ocurrencia del valor especificado o -1 si no se encuentra dicho valor. La búsqueda comienza desde indiceDesde.

El método lastIndexOf() devuelve la posicion (indice) de la última ocurrencia en la que se encuentra el valor busquedo.

        
          // cadena.indexOf(valorBusqueda[, indiceDesde])

          let banda = "Catupecu Machu";
          console.log(banda.indexOf("pe")); // true 
          console.log( banda.indexOf( "z" ) ); // -1
          console.log( banda.search( "Catu" ) ); // 0

Eliminando espacios

El método trim() elimina los espacios en blanco en ambos extremos del string. Los espacios en blanco en este contexto, son todos los caracteres sin contenido (espacio, tabulación, etc.) y todos los caracteres de nuevas lineas.

El método trimStart() elimina los espacios en blanco del principio de una cadena. trimLeft() es un alias de este método.

El método trimEnd() elimina los espacios en blanco al final de una cadena de carácteres.

        
          let banda = "  Catupecu Machu  ";
          console.log( `#${banda.trim()}#` ); // #Catupecu Machu#
          console.log( `#${banda.trimStart()}#` ); // #Catupecu Machu  #
          console.log( `#${banda.trimEnd()}#` ); // #  Catupecu Machu#

Dividir cadenas de texto

El método split() divide una cadena en una lista ordenada de subcadenas. La división se hace buscando un patrón.

        
          let banda = "Catupecu Machu";
          const words = banda.split(' ');
          console.log(words[0]); // Catupecu
          console.log(words[1]); // Machu
          console.log(words); // ['Catupecu', 'Machu']

Otros métodos y propiedades

Existen muchas métodos y propiedades como replace, match, repeat, slace, etcetera para trabajar con cadenas. Podemos encontrarlos en la ducumentación de MDN

Typo Boolean

El tipo boolean tiene sólo dos valores posibles: true y false. Este tipo se utiliza comúnmente para almacenar valores de sí/no: true significa “sí, correcto, verdadero”, y false significa “no, incorrecto, falso”. 

        
          let nameFieldChecked = true; // sí, el campo name está marcado
          let ageFieldChecked = false; // no, el campo age no está marcado

Los valores booleanos también son el resultado de comparaciones:

        
          let isGreater = 4 > 1;
          alert( isGreater ); // verdadero (el resultado de la comparación es "sí")

Tipo null

En JavaScript, null no es una “referencia a un objeto inexistente” o un “puntero nulo” como en otros lenguajes. Es sólo un valor especial que representa “nada”, “vacío” o “valor desconocido”.

        
          let age = null;

Tipo undefined

El significado de undefined es “valor no asignado”. Si una variable es declarada pero no asignada, entonces su valor es undefined:

        
          let age;
          alert(age); // muestra "undefined"

          // es posible asignar undefined a cualquier variable:
          let age = 100;
          age = undefined;
          alert(age); // "undefined"
Normalmente, usamos null para asignar un valor “vacío” o “desconocido” a una variable, mientras undefined es un valor inicial reservado para cosas que no han sido asignadas.

Tipos Object y Symbol

El tipo object (objeto) es especial. Todos los demás tipos se llaman “primitivos” porque sus valores pueden contener una sola cosa (ya sea una cadena, un número, etc). Por el contrario, los objetos se utilizan para almacenar colecciones de datos y entidades más complejas.

Más adelante veremos el tipo de datos Object en profundida.

El tipo symbol (símbolo) se utiliza para crear identificadores únicos para los objetos. Tenemos que mencionarlo aquí para una mayor integridad, pero es mejor estudiar este tipo después de los objetos.

El operador typeof

El operador typeof devuelve el tipo de dato. La llamada a typeof x devuelve una cadena con el nombre del tipo: 

        
          typeof undefined // "undefined"
          typeof 0 // "number"
          typeof 10n // "bigint"
          typeof true // "boolean"
          typeof "foo" // "string"
          typeof Symbol("id") // "symbol"
          typeof Math // "object"  (1)
          typeof null // "object"  (2)
          typeof alert // "function"  (3)
  • Math es un objeto incorporado que proporciona operaciones matemáticas. Lo aprenderemos en el capítulo Números. Aquí sólo sirve como ejemplo de un objeto.
  • El resultado de typeof null es "object". Esto está oficialmente reconocido como un error de comportamiento de typeof que proviene de los primeros días de JavaScript y se mantiene por compatibilidad. Definitivamente null no es un objeto. Es un valor especial con un tipo propio separado.
  • El resultado de typeof alert es "function" porque alert es una función. Estudiaremos las funciones en los próximos capítulos donde veremos que no hay ningún tipo especial “function” en JavaScript. Las funciones pertenecen al tipo objeto. Pero typeof las trata de manera diferente, devolviendo function. Además proviene de los primeros días de JavaScript. Técnicamente dicho comportamiento es incorrecto pero puede ser conveniente en la práctica.
