Intoducción AJAX

AJAX (JavaScript Asíncrono y XML) es un término que describe dos capacidades de los navegadores: la posibilidad de hacer peticiones al servidor sin tener que volver a cargar la página y la posibilidad de analizar y trabajar con documentos XML.

Aunque aún existen implementaciones para intercambio de datos en XML, actualmente utilizamos Json. Inicialmente, XML era el formato de intercambio de información entre sistemas, ya que los distintos lenguajes de programación poseen la capicidad de leer este formato.

JavaScript Asíncrono + XML (AJAX) no es una tecnología por sí misma, es un término que describe un nuevo modo de utilizar conjuntamente varias tecnologías. Esto incluye: HTML, CSS, JavaScript, DOM, XML, XSLT, y el objeto XMLHttpRequest. Cuando estas tecnologías se combinan en un modelo AJAX, es posible lograr aplicaciones web capaces de actualizarse continuamente sin tener que volver a cargar la página completa. Esto crea aplicaciones más rápidas y con mejor respuesta a las acciones del usuario.

Tadicionalmente, cuando solicitamos un recurso se genera un petición (request) desde el navegador al servidor donde se encuentre alojado el recurso. De acuerdo al recurso solicitado, el servidor resuelve obteniendo datos de una base de datos, consultando microservicios, etc; y envía la respuesta (response) compuesta por archivos html, css, JavaScript, imágenes al navegador.

Esto quiere decir que cada vez que solicitamos un recurso (navegamos a una página), se recarga el navegador.

Ajax nace para mejorar el rendimiento y la experiencia del usuario, ubicándose como un intermediario entre la interfax del usuario y el servidor. De esta manera, no necesitamos recargar el recurso, solo se actualizan los datos necesisarios.

Ajax no es una tecnología en sí mismo. Se trata de varias tecnologías independientes que se unen:

  • HTML y CSS, para crear una presentación basada en estándares.
  • DOM, para la interacción y manipulación dinámica de la presentación.
  • HTML, XML y JSON, para el intercambio y la manipulación de información.
  • XMLHttpRequest o Fetch, para el intercambio asíncrono de información.
  • JavaScript, para unir todas las demás tecnologías.

Métodos nativos para implementar Ajax

  • ActiveXObject. Es para Internet Explorer 8. Está obsoleto.
  • XMLHttpRequest
  • API Fetch

XMLHttpRequest

XMLHttpRequest es un objeto JavaScript que creado por Microsoft y adoptado por el resto de los navegadores. Actualmente es un estándar de la W3C. Proporciona una forma fácil de obtener información de una URL sin tener que recargar la página completa. Una página web puede actualizar sólo una parte de la página sin interrumpir lo que el usuario está haciendo.

A pesar de su nombre, XMLHttpRequest puede ser usado para recibir cualquier tipo de dato, no solo XML

API Fetch

La API Fetch proporciona una interfaz para recuperar recursos. La nueva API fecth es más potente y flexible que XMLHttpRequest.

Fetch ofrece una definición genérica de los objetos Request y Response (y otras cosas relacionadas con las solicitudes de red). También proporciona una definición para conceptos relacionados, como CORS y la semántica de encabezado HTTP origen

Librerías externas para implementar ajax

  • jQuery.ajax. Fué una de las más utilizadas, pero a medida que el estandar JavaScript incorpora nuevas características va quedando en desuso. Cuando los distintos navegadores trabajaban de distinta manera, jQuery nos permitía abstraernos del navegador. jQuery.ajax se encargaba de detectar el navegador, si se trataba de IE traducía la petición a ActiveXObject, y si era otro navegador traducía a XMLHttpRequest.
  • Axios. Es muy popular y se basa en promesas. Nos permite interceptar solicitudes y respuestas, transformar datos de solicitud y respuesta, cancelar solicitudes, transformaciones automáticas para datos JSON, compatibilidad del lado del cliente para protección contra XSRF.
  • etc

Estado de las peticiones

Una petición Ajax tendrá 5 estados.

  • READY_STATE_UNINITIALIZED = 0, el cliente se comunica con el servidor.
  • READY_STATE_LOADING = 1, se envían los datos al servidor.
  • READY_STATE_LOADED = 2, el servidor respondió al cliente pero el cliente aún no procesa la información.
  • READY_STATE_INTERACTIVE = 3, el motor de JavaScript accede a los datos recibidos desde el servidor.
  • READY_STATE_COMPLETE = 4, en este punto ya tenemos los datos disponibles para manipularlos.

De la interacción entre el cliente y el servidor surgen distintos códigos de estado de respuesta HTTP.

  • Respuestas informativas (100-199)
  • Respuestas satisfactorias (200-299)
  • Redirecciones (300-399)
  • Errores de los clientes (400-499)
  • y errores de los servidores (500-599)
