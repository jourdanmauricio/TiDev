Asincronismo en JavaScript

Temporizadores

A través de los temporizadores podemos ejecutar acciones a posteriori. En JavaScript existen 2: setTimeOut() y setInterval().

setTimeOut()

El método setTimeout() establece un temporizador que ejecuta una función o una porción de código después de que transcurre un tiempo establecido.

        
          console.log('Inicio setTimeout');
          setTimeout(() => {
            console.log('Ejecutando un setTimeOut, se ejecuta una sola vez');
          }, 2000);
          console.log('Fin setTimeout');

El método clearTimeOut() cancela setTimeOut(). Para que funcione el setTimeOut debe estar almacenado en una variable. En el siguiente ejemplo inicia el temporizador pero es cancelado inmediatamente. El console.log de la función nunca se ejecuta.

        
          let temporizador = setTimeout(() => {
            console.log('Ejecutando un setTimeOut');
            console.log('Pero lo estamos cancelando');
          }, 2000);

          clearTimeOut(temporizador);

setInterval()

El método setInterval() ejecuta una función o un fragmento de código de forma repetitiva cada vez que termina el periodo de tiempo determinado. Devuelve un ID de proceso. 

        
          console.log('Inicio setInterval');
          setInterval(() => {
            console.log('Ejecutando un setInteval');
            console.log('Se ejecuta indefinidamente cada cierto intervalo de tiempo');
          }, 3000);

El método clearInterval() cancela el setInterval (también debe estar almecenado en una variable). 

        
          let temporizador = setInterval(() => {
            console.log('Ejecutando un setInteval');
            console.log('Se ejecutaría indefinidamente cada cierto intervalo de tiempo');
            console.log('Pero lo estamos cancelando');
          }, 3000);

          clearInterval(temporizador);
