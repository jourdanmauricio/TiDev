Posicionamiento en CSS

Las propiedades FLOAT y POSITION nos permiten modificar la posición natural de cualquier elemento del documento HTML.

El navegador coloca cada elemento teniendo en cuenta el orden en el que aparece en el documento y su tipo de visualización dependiendo si es un elemento de línea o de bloque.

Con FLOAT y POSITION podemos modificar este comportamiento.

Elementos Flotantes

  • float: Convierte un elemento en flotante desplazándolo hasta la zona más a la izquierda o más a la derecha de la posición en la que originalmente se encontraba
  • clear: Limpia la flotación (left, right, both)

Tipos de Posicionamiento

  • static (default)
  • relative
  • absolute
  • fixed
  • sticky

Para mover los elementos posicionados se activan las propiedades:

  • top (vertical - eje Y)
  • bottom (vertical - eje Y)
  • left (horizontal - eje X)
  • right (horizontal - eje X)
  • z-index (profundidad - eje Z)

Las propiedades top y left van a tener preferencia por sobre bottom y right respectivamente. Estás 5 propiedades no funcionan con el valor de static.

Posicionamiento static

static: Es el valor por defecto, significa que el elemento respeta el flujo del documento HTML, por lo que no se puede mover ni posicionar.

          
            <div class="static">Caja Estática</div>
          
        
          
            .static {
              background-color: yellow;
              position: static;
              top: 10px;
              left: 10px;
            }
Caja Estática

A pesar de asignar las propiedades top: 10px y left: 10px; la caja no se mueve porque es una caja estática. No incluimos la propiedad position: static porque el valor por default de las cajas.

Posicionamiento relative

relative: El elemento conserva sus dimensiones y posición original en el flujo del documento. Si se mueve toma como referencia su posición original.

Un error común al asignar la propiedad top: 50px; es pensar que se moverá 50px hacia arriba. El valor top no indica hacia donde debe desplazarse sino que margen tomará como referencia para el despazamiento. En este caso, se moverá 50px desde el margen top hacia abajo. Con left: 50px; la caja se despazará a la derecha. Con right la caja se moverá a la izquierda. Y con bottom subirá.

También se debe considerar que el elemento se desplaza pero "conserva" su espacio original en el documento. 

          
            <div class="relative">Caja Relativa</div>
          
        
          
            .relative {
              background-color: turquoise;
              position: relative;
              width: 300px;
              height: 50px;
              top: -50px;
              left: 200px;
              bottom: 50px;
              right: 50px;
            }
Caja Relativa

Si utilizamos la propiedad bottom y top, bottom será ignorada. Lo mismo ocurre con left y right. Right será ignorada.

Posicionamiento absolute

absolute: El elemento pierde sus dimensiones y posición original en el flujo del documento. Si se mueve puede tomar como referencia 2 elementos:

  • El primer ancestro con posicionamiento relativo
  • Si no encuentra un ancestro relativo, se mueve respecto del documento HTML
          
            <div class="absolute">Caja Absoluta</div>
            <section class="relative-parent">
              <div class="absolute-child">
                Caja Absoluta con Ancestro Relativo
              </div>
            </section>
          
        
          
            .absolute {
              background-color: tomato;
              position: absolute;
              width: 200px;
              height: 50px;
              /* top: 0;
              left: 0; */
              /* top: 100px;
              left: 100px; */
              right: 0;
              bottom: 0;
            }
            .relative-parent {
              background-color: khaki;
              position: relative;
              width: 600px;
              height: 50px;
              top: 50px;
              left: 50px;
            }
Caja Absoluta
Caja Absoluta con Ancestro Relativo

Cuando movemos un elemento que posee la propiedad absolute, el navegador lo posicionará en relación al ancestro que posea la propiedad relative. En caso de no encontrar ninguno lo posicionará según el elemento html. Por otro lado, el elemento perderá su posición original en el documento y su tamaño. Por eso es importante asignarle width y height.

Posicionamiento fixed

El elemento pierde sus dimensiones y posición original en el flujo del documento. Si se mueve toma como referencia el docuemento HTML y queda fijo en la posición, cuando el scroll se mueva, el elemento no lo hará, queda fijo.

          
            <div class="fixed">Caja fija</div>
          
        
          
            .fixed {
              background-color: lightgreen;
              position: fixed;
              width: 300px;
              height: 50px;
              /* right: 20px;
              bottom: 10px; */
              top: 30px;
              left: 450px;
            }
Caja fija en section

Como el elemento queda fijo en la pantalla es utilizado para posicionar menus en la parte superior de la pantalla.

Posicionamiento sticky

Caja sticky

Sticky: es una combinación entre posicionamiento relative y fixed, en este posicionamiento las propiedades top, right, bottom y left no mueven el elemento.

Sirve como punto de referencia, mientras el scroll no llegue a ese valor, el elemento se comporta como relative cuando llega al elemento se comporta como fixed.

Para que este compartamiento funcione el elemento sticky debe ser hijo directo del body o su elemento contenedor debe tener dimensiones definidas y sólo será sticky dentro de las dimensiones de su contenedor padre.

          
            <div class="sticky-child">Caja sticky</div>
          
        
          
            .sticky {
              background-color: lightskyblue;
              position: sticky;
              top: 0px;
            }
            .sticky-parent {
              border: thin solid black;
              width: 100%;
              height: 600px;
            }
            .sticky-child {
              background-color: lightpink;
              position: sticky;
              top: 20px;
            }

Para demostrar el comporamiento de sticky dentro de un padre article asignamos dimensiones al article y border (para visiualizarlo) y definimos que el elemento sticky se fije en 20px;. En cuanto el scroll llega al final del padre, el elemento deja de ser fixed y pasa a relative.

z-index

Nos permite controlar la profundidad de los elementos posicionados, su valor por defecto es auto, acepta números positivos, negativos y cero.

A mayor valor, el elemento estará mas al frente, a menor valor, más al fondo. Conceptualmente podemos asociarlo a números de capas en diseño.

Un elemento padre nunca podrá estar sobre sus elementos hijos, sin embargo los elementos hijos si pueden asignándoles un z-index negativo, pero el elemento padre no debe tener asigano un z-index. 

          
            <div class="z-index1">Probando z-index caja 1</div>
            <div class="z-index2">Probando z-index caja 2</div>
          
        
          
            .z-index1,
            .z-index2 {
              background-color: mediumaquamarine;
              border: thin solid black;
              width: 200px;
              height: 200px;
            }
            .z-index1 {
              position: relative;
              z-index: 5;
            }
            .z-index2 {
              position: relative;
              top: -100px;
              left: 100px;
              z-index: 5;
            }
Probando z-index caja 1
Probando z-index caja 2

La propiedad z-index funciona sobre elementos que posean posicionamiento diferente a static.

Cuando el z-index de los elementos es auto la preferencia se define según el flujo html.

Cuando utilizamos la propiedad z-index es recomendable no trabajar con valores consecutivos por si luego tenemos que insertar elementos intermedios.
