Feature Queries

Las Feature queries son reglas CSS que le preguntan al navegador si soportan o no una determinada regla o valor. Cuando se incorporan nuevas características y las queremos utilizar, podemos preguntar al navegador si se encuentra soportada e implementarlar o utilizar otras características.

Ejemplo, a partir del año 2012 podiamos maquetar en Flexblox pero en 2017 se incorporó la maquetación con Grid. Actualmente, todos los navegadores soportan Grid, pero inicialmente teníamos que definir una media query preguntanso si el navegador soportaba Grid para implementarlo. Si no lo soportaba maquetabamos con Flexbox.

Las feature queries nos permiten implementar nuevas características y mantener la compatibilidad hacia atrás a través del operador not. También podemos utilizar los operadores or y and como condicionales. 

        
          /* Preguntamos si el navegador soporta la propiedad grid con el valor display */
          @supports (display: grid) {
            /* 
            Código CSS que aplicará cuando cumpla con al feature query
            Clases para grid
            */
          }
          @supports not (display: grid) {
            /* 
            Código CSS que aplicará cuando cumpla con al feature query
            Clases para Flexbox
            */
          }
        
      
        
          @supports (display: grid) and (grid-template-columns: subgrid) {
            html {
              background-color: darkslateblue;
              color: lightseagreen;
            }
          }
          @supports (display: grid) or (grid-template-columns: subgrid) {
            html {
              background-color: white;
              color: black;
            }
          }
