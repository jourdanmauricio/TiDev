Details

Este elemeto se confirma por tres etiquetas: details, summary y el contenido que puede ser texto, imágenes,etc. Ejemplo:

          
            <details>
              <summary>Titulo del acordeón</summary>
              <article>
                <h3>Contenido del acordeón</h3>
                <p>Lorem ipsum dolor sit amet consectetur, adipisicing elit. Eveniet suscipit corporis a consequuntur
                dolores ipsam reiciendis, perspiciatis corrupti perferendis sint porro eligendi. Reiciendis voluptas dicta
                quibusdam. Ex molestias ut placeat?</p>
                <img class="redu" src="/assets/images/imagen-png.png" alt="Imagen de pinguino">
              </article>
            </details>

Ventanas modales (experimental)

El elemento HTML dialog representa un cuadro de diálogo, como una alerta o una subventana. El atributo open indica que el cuadro de diálogo está activo y se puede interactuar con él. Cuando el atributo open no está configurado, el cuadro de diálogo no debe mostrarse al usuario. Se recomienda utilizar el método .show() o .showModal() para representar cuadros de diálogo, en lugar del atributo open.

El siguiente ejemplo generará un diálogo modal abierto. El botón "Aceptar" permite cerrar el cuadro de diálogo cuando se activa. Es importante proporcionar un mecanismo para cerrar un diálogo dentro del elemento de diálogo. Por ejemplo, la tecla Esc no cierra los cuadros de diálogo sin modo de forma predeterminada, ni se puede suponer que un usuario tendrá acceso a un teclado físico (por ejemplo, alguien que usa un dispositivo de pantalla táctil sin acceso a un teclado).
