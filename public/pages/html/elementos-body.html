Elementos del cuerpo Body

El cuerpo es la segunda sección de la estructura del documento HTML y se delimita por las etiquetas <BODY>...</BODY>. Dentro de estas etiquetas se encuentra toda la información que se renderizará en el navegador. Contiene elementos como: encabezados, textos, tablas, imágenes, animaciones, etc; que contienen sus propios atributos.

El elemento head de un documento HTML es la parte que no se muestra en el navegador cuando se carga la página. La función de la cabecera es contener los metadatos del documento.

SEO: Solo se debe definir un elemento h1 por página. Los robots de los buscadores buscan un título principal por cada página. Podemos incluir tantos h2, h3, h4, h5 o h6 como deseemos pero debemos respetar el orden. Para definir correctamente la titulación considiremos la redacción de un artículo o de un capítulo de un libro.

Etiquetas semánticas estructurales

Una de las ventajas de HTML5 son las etiquetas semánticos, que nos permiten definir las distintas divisiones de una página web.

En las versiones anteriores se utilizaba la etiqueta <div> para realizar las divisiones de las distintas secciones de una página web, pero se recomienda utilizar los elementos semánticos ya que describen su propósito. Siempre que podamos debemos intentar utilizar las etiquetas smanicas en lugar de DIV. 

<header>

Define el encabezado de una página web o sección.

<nav>

Define una sección con enlaces de navegación.


  <header>
  <h1>HTML Reference</h1>
  <nav>
      <a>Home</a>
      <a>About</a>
      <a>Contact</a>
    </nav>
  </header>
  
<section>

Define una sección dentro de una página web.


  <section>
    <h2>Section title</h2>
    <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Donec viverra nec nulla vitae mollis.</p>
  </section>
  
<aside>

Define un bloque de contenido relacionado con el contenido principal. Generalmente se muestra como una barra lateral.

<footer>

Define el pie de página de una página web o sección.

Etiquetas de bloque vs etiquetas de línea

En Html existen eqtiquetas que de acuerdo a su comportamiento son de bloque o de línea. La etiqueta de bloque más respresentativa es la etiqueta DIV y la etiqueta de línea mas utilizada es la etiqueta SPAN. 

<div>

Define un bloque genérico de contenido, que no tiene ningún valor semántico. Se utiliza para elementos de diseño como contenedores.

<span>

Define un genérico de contenido en línea, que no tiene ningún valor semántico.

Una etiqueta de bloque genera un salto de línea con sus etiquetas hermanas, tanto arriba como abajo. Es decir, ocupa todo el ancho disponible.

<div style="background-color:pink;"> 
    Este es un bloque DIV. Pintamos el fondo de amarillo para que se vea su magnitud real. Vemos como ocupa el 100% del ancho aunque no lo necesite. 
  </div>

Son etiquetas de bloque: los encabezados, los párrafos, div, header, main, footer, etc. Son etiquetas de línea: em, etrong, i, mark, b, blockout, las que aplican a texto, a, etc
