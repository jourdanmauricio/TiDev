Contenedores Flexibles

Todos los elementos de nuestra interfaz tienen que ser flexibles y responsivos.

Cuando asignamos medidas absolutas y el elemento es mayor a la pantalla del dispositivo aparecerá la barra de scroll vertical.

        
          <div class="box"></div>
        
      
        
          .box {
            width: 300px;
            height: 300px;
          }

Una caja flexible posee las dimensiones definidas a través de unidades relativas. Si utilizamos la medida en porcentajes, al ancho relativo será con respecto a su contendor padre. Para utilizar altura en porcentajes tenemos que considerar que su contenedor padre tenga definidas la medida de altura (height).

Podemos utilizar el viewport como unidad de medida. En este caso, utilizamos el 20% de la altura de la pantalla.

        
          <div class="box"></div>
        
      
        
          .box-flexible {
            width: 50%;
            height: 20vh;
          }

Expresar el ancho en medidas relativas permite que el elemento se adapte (sea flexible) a los distintos dispositivos. En el caso anterior siempre ocupará el 50% del ancho.

También, existen propiedades CSS que nos permiten crear contenedores flexibles aunque utilicemos unidades absolutas como los pixeles.

Propiedades max-width, min-width, max-height y min-height

En lugar de definir una alto y ancho podemos definir máximos y mínimos. No definimos el ancho base, sino la máxima anchura que tomará el elemento. En el ejemplo el elemento no crecerá más allá de 960px y no decrecerá menos de 280px. A pesar de no utilizar unidades relativas conseguimos un contenedor flexible.

        
          <div class="box"></div>
        
      
        
          .box-flexible {
            width: 50%;
            height: 20vh;
          }

La altura siempre depende del contenido. En este caso, como el elemento no posee contenido toma la altura mínima (280px). Para verificar que la altura no supere los 480px creamos un nuevo contenedor con suficiente contenido para que supere la altura máxima.

Los cinco párrafos no entran en los 280px (min-height), por lo que el elemento crece verticalmente. Si reducimos la pantalla el contenedor crecerá hasta 480px (max-height), a patir de esa altura el contenido del último párrafo desbordará el contenedor.

Tamaños fijos VS Tamaños máximos y mínimos

Si en el contenedor definimos mínimos y máximos pero además definimos medidas width y height específicas en pixeles aplicará las medidas específicas ignorando los máximos y mínimos.

        
          <div class="box-flexible-3">
        
      
        
          .box-flexible-3 {
            max-width: 960px;
            min-width: 280px;
            max-height: 480px;
            min-height: 280px;
            width: 300px;
            height: 300px;
          }

Si definimos mínimos y máximos, y además width y height pero en unidades relativas como porcentajes o unidades viewport se aplicará el width y height pero se respetarán los mínimos y máximos.

        
          <div class="box-flexible-4">
        
      
        
          .box-flexible-4 {
            max-width: 960px;
            min-width: 280px;
            max-height: 480px;
            min-height: 280px;
            width: 50%;
            height: 20vh;
          }

Conclusión: las propiedades widht y height tienen mayor jerarquía que los mínimos y máximos. Si las unidades widht y height se expresan en unidades absolutas se ignorarán los mínimos y máximos. En cambio, si se expresan en unidades relativas serán consideradas.No utilizar máximos, mínimos y widht y height con unidades absolutas.
