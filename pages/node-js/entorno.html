Configurar las variables de entorno en Node.js

Las variables de entorno son una forma de llevar información de afuera a nuestro software. Se utiliza para definir parámetros sencillos de configuración de los programas. Ejemplo: podemos enviar el modo de ejecución del ambiente sin necesidad de modificar el código fuente, la url de una API que consulte nuestra aplicación, o información sensible como un token, o claves de acceso que no podemos hardcodear en nuestro código. El código no debe guardar ningún tipo de credenciales o valores que vayan a cambiar. Para esto utilizamos las variables de entorno.

        
          /* variables_entorno.js */
          let name = process.env.NAME;
          console.log('Hola ' + name + '!');
        
      
        
          /* Ejecutamos */
          NOMBRE = 'Mauricio' 
          node variables_entorno.js

JavaScript tiene la opción de utilizar valores por defecto: 

        
          /* variables_entorno.js */
          let name = process.env.NAME || 'invitado';
          console.log('Hola ' + name + '!');

Si queremos pasar más de una variable:

        
          /* variables_entorno.js */
          let name = process.env.NAME || 'invitado';
          let web = process.env.WEB || 'sin web';

          console.log('Hola ' + name + '!');
          console.log('Mi página web es: ' + web);
        
      
        
          /* Ejecutamos */
          NOMBRE = 'Mauricio' 
          WEB='mi_web.com' 
          node variables_entorno.js
Por convención las variables de entorno se escriben en mayúsculas y en caso de tener mas de una palabra se separan con guión bajo '_'.
