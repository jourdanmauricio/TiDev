EventLoop: asíncrona por diseño

Evento loop es un bucle que gestiona de forma asíncrona y automática de todos los eventos de tu aplicación. El blucle no se bloquea, recibe peticiones en todo momento, por esto es altamente concurrente. Todos las tareas que reciba el bucle se ejecutarán aparte para que puede seguir recibiendo peticiones. Es altamente concurrente.

El bucle de eventos (event loop) es un bucle infinito que recibe eventos desde la cola de eventos (event queue). La cola de eventos contiene todas las instrucciones de nuestro código, que se envían una a una al bucle de eventos. El bucle de eventos procesará los eventos. Si puede, resuelve la tarea, si la tarea es asíncrona la envía al Thread pool.

Las operaciones de bases de datos, que trabajan con archivos, las operaciones muy lentas no bloquean el hilo principal, el event loop continúa funcionando y cada hilo se encarga de gestionar sus procesos lentos (asíncronos).

Esta característica diferencia a Node js de otros lenguajes de servidor como PHP, Phyton, o Java. Algunos lenguajes son síncronos y las operaciones se ejecutan una detrás de otra, y cuanto más complejas son las operaciones más tiempo demorarán las demás. Node js permite ejecutar tareas en paralelo.

El thread pool levanta un hilo nuevo en el procesador por cada tarea que tenga que ejecutar. Cuando la tarea finaliza el hilo dispara un evento al thread pool que será enviado al event loop. Y, el event loop evaluará si devolverlo al event queue porque puede haber algún proceso escuchando para generar una nueva función.

Es importante entender el concepto de asincronía. Puede ocurrir que finalice una instrucción antes que otra que era anterior. Si una tarea es compleja se delega, y continúa la ejecución.

Las desventajas se encuentran relacionadas a seguridad, gestión de errores, etc.
