Proxies

El objeto Proxy permite crear un intermediario para otro objeto, el cual puede interceptar y redefinir operaciones fundamentales para dicho objeto.

Podemos asimilarlo a la instanciación de clases, pero en lugar de utilizar una clase utilizaremos un objeto literal. El proxy recibe el objeto literal, genera una copia y nos permite realizar operaciones como validar propiedades dentro de la copia que se esta generando. Es decir, tendremos un medio de vinculación entre el objeto original y el nuevo objeto generado.

Este mecanismo se administra a través de un objeto especial, que también recibe el proxy, llamado handler.

        
          // Objeto base
          const person = {
            nombre: "",
            apellido: "",
            edad: 0
          }
  
          // handler
          const manejador = {
            set( obj, prop, valor ) {
              // no aplicamos validaciones
              /* obj[prop] = valor; */
  
              // Verificamos que la propiedad exista en el objeto original person
              /* if (object.keys(obj).indexOf(prop) === -1) {
                return console.error(`La propiedad '${prop}' no existe en person.`);
              }
              obj[prop] = valor; */
  
              // Validamos que nombre y apellido solo tengan letras y espacio en blanco
              if (
                ( prop === 'nombre' || prop === 'apellido' )
                && !( /^[A-Za-zÑñÁáÉéÍíÓóÚú\s]+$/g.test( valor ) )
              ) {
                return console.error( `La propiedad '${prop}' solo acepta letras y espacios.` );
              }
              obj[prop] = valor;
            }
          }
  
          const mau = new Proxy( person, manejador );
          console.log( mau );
  
          mau.nombre = 'Mauricio';
          mau.apellido = 'Jourdán';
          mau.edad = 47;
          mau.twitter = '@jourdanmau'
          console.log( "mau", mau );
  
          console.log( person );
          mau.apellido = 'Jourdán74';
