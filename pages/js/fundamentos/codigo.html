Estructura del código

Sentencias

Las sentencias son construcciones sintácticas y comandos que realizan acciones. Podemos tener tantas sentencias en nuestro código como queramos, las cuales se pueden separar con un punto y coma o, como generalmente se realiza, escribir en líneas separadas.

Se puede omitir un punto y coma en la mayoría de los casos cuando existe un salto de línea.

Comentarios

En ocasiones, cuando es código es coplejo, se pueden agregar comentarios que serán ignorados por el motor del navegador.

Los comentarios de una línea comienzan por //, mientras que los comentarios de varias líneas comienzan con /* y terminan con */.

        
          // Este es un comentario de una línea
          alert('Ejemplos de comentarios!'); 
          console.log('Ejemplos de comentarios!');
          /* Este es un comentatio
          de más de una línea */

Nomenclatura

Por convención se utilizan las siguientes reglas para nombrar variables, constantes, funciones, archivos, clases, objetos, etc.

  • Los identificadores (nombres de variables, funciones, clases, archivo, etc) deben comenzar con:
    • una letra
    • un signo $
    • un guión bajo _
    • No deben comenzar con números o caracteres especiales
    • Aunque se encuentra permitido no utilizar caracteres fuera del idioma inglés como ñ o acentos. De esta manera evitaremos inconvenientes.
    • No utilizar palabras clave del lenguaje.
  • Utilizar UPPER_CASE (mayúsculas y guiones bajos para separar las palabras) en:
    • Constantes. 
                       
                  const PI = 3.141592653589793;
                  const BACKGROUD_COLOR = "blue";
  • Utilizar snake_case (las palabras se separan por guión bajo y se escriben en minúscula) en:
    • Archvivos. 
                       
                  nombre_archivo.js;
  • Utilizar UpperCamelCase (Cada palabra comienza con mayúsculas) en:
    • Clases. 
                       
                  class  CategoriaProducto {}
  • Utilizar lowerCamelCase (mayúsculas en primera letra de cada palabra, excepto en la primera) en:
    • Objetos. 
                       
                  const myObject = {
                    myName = "Mauricio"
                    myAge = 35
                  }
    • Primitivos. 
                       
                  let unaCadena = "esta es una cadena";
                  let unNumero = 35
    • Funciones. 
                       
                  function myFunction (text) {
                    alert(`Mensaje desde una ${text}`)
                  }
                  myFunction("función");
    • Instancias. 
                       
                  const ajax = new XMLHttpRequest()
