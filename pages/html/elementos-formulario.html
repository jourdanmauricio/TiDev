El atributo name es obligatorio para enviar información, ya que se recuperará a través de este atributo.

El imput de tipo email es semantico. Espera que la información que enviemos contenga un @ y los dispositivos agregarán al teclado tactil algunos botones útiles como @ y .com

Lo mismo ocurre con el tipo number. No solo que no permite el ingreso de texto sino que en dispositivos táctiles cambia el teclado por uno solo numérico.

El input de tipo password no muestra el texto ingresado.

El tipo textarea nos permite ingresar más texto y podemos definir la cantidad de filas y columnas.

La eqtiqueta select nos permite seleccionar entre varias opciones. El atributo multiple nos permite seleccionar varias opciones.

El atributo disabled transforma el input en un elemento de solo lectura y la propiedad disabled genera el efecto visual de inhabilitado.

          
            <input type="text" value="Mauricio" readonly disabled>

Otro atributo interesante es el placeholder. Nos ayuda a guiar al usuario indicando el tipo de texto a ingresar. En cuanto el usuario comienza a escribir el placeholder desaparece.

          
            <input type="text" placeholder="Ingresa tu nombre">

El formulario

El formulario se representa a través de la etiqueta form, es un elemento de bloque. El atributo action determina hacia donde se enviará la información. Dentro de los formularios los botones de tipo submit y reset cobran relevancia. Reset vuelve los elementos a su valor por defecto, miestras que submit enviará el formulario a la dirección indicada en el atributo action del formulario.

Agregamos la etiqueta br porque los elementos de tipo input son inline. Así agregamos un salto de línea.

    
      <h4>Ejemplo formulario login</h4>
      <form action="#formulario">
      <input type="text" name="usuario" placeholder="Ingresa tu usuario"><br>
      <input type="password" name="password"><br>
      <input type="reset" value="Limpiar">
      <input type="submit">
      </form>

Ejemplo formulario login



En este caso configuramos el action para que vuelva a la misma sección del documento. Luego veremos como procesamos la información enviada, pero a priori si completamos los campos usuario y password y presionamos enviar podemos ver cómo se envían los datos como parámetros a trapés de la url. Aquí vemos el atributo name en acción.

Existen dos métodos de envío. El método GET es el que se encuentra establecido por defecto, por eso no configuramos el atributo method. La segunda manera de enviar los datos del formulario es a través del valor POST en el atributo method. La diferencia es que los datos no se envían como parte de la url, sino que se envían ocultos en la cabecera de la petición html. Veremos con mayor profundidad el paso de parámetros en la seccion de JavaScript.

    
      <h4>Ejemplo formulario login</h4>
      <form action="#formulario" method="POST">  <!--method="GET"-->
      <input type="text" name="usuario" placeholder="Ingresa tu usuario"><br>
      <input type="password" name="password"><br>
      <input type="reset" value="Limpiar">
      <input type="submit">
      </form>

Vemos que al utilizar los formularios el navegador nos ofrece datos que se han utilizado antes. Si deseamos evitarlo podemos colocar el atributo autocomplete en off. El valor por defecto es on.

Adicionalmente podemos agregar etiquetas label para indicar a qué refiere el input. En el parámetro for se coloca el id (identificador) del input. De esta manera, si el usuario realiza un click sobre el label el foco pasará al input. Que el atributo name y id posean el mismo valor es casual, podrían ser diferentes. Es información completamente diferente, recordemos que name se utiliza para recuperar la información una vez enviado el formulario y el atributo id entre otras cosas se utiliza para hacer referencia desde el label.

Otro atributo importante es required. Le indica al navegador que realice una validación al momento de enviar el formulario. Si un input posee el atributo required y no fue completado nos mostrará un mensaje indicando la obligatoriedad del input y no enviará el formulario.

También podemos realizar validaciones personalizadas a través del atributo pattern que utiliza expresiones regulares. Más adelante veremos cómo utilizar las expresiones regulares. Por el momento veremos una expresión sencilla que solo admita letras. Las expresiones regulares comienzas con ^ y finalizan con $. En el atributo title podemos indicar el mensaje que aparecerá cuando el texto ingresado por el usuario no pase la validación.

Finalmente, comentar que si no deseamos que ningún input del formulario se autocomplete podemos agregar el atributo autocomplete="off" al form. El atributo se puede utilizar a nivel formulario o por cada input.

          
            <h4>Ejemplo formulario login</h4>
            <form action="#formulario" method="POST">  <!--method="GET"-->
              <label for="usuario">Usuario</label>
              <input type="text" name="usuario" id="usuario" pattern="^[A-Za-z]+$" title="Ingresa solo letras" required><br><br>
              <label for="password">Password</label>
              <input type="password" name="password" id="password" required><br><br>
              <input type="reset" value="Limpiar">
              <input type="submit">
            </form>

Selects, Radios y Checkboxs

Las etiquetas select muestran por defecto el contenido del primer option que tengan. Por ese motivo solemos ingresar una opción que indique "Seleccione..." y cuyo value sea igual a "". Si deseamos que la opcion seleccionada no sea la primera podemos agregar el atributo selected a la opción que deseemos.

En el caso de los selects el valor que se enviará es el value de la opción seleccionada.

Las etiquetas de tipo radio deben tener el mismo nombre para que solo podamos selecionar una alternativa. El navegador interpreta que son un grupo y por comportamiento del radio button al seleccionar una alternativa desmarcará las otras.

También es importante destacar que el valor que se enviará con el formulario es el valor del atributo value.

El atributo selected también aplica a los redio buttons pero se denomina checked. También acepta el atributo required, con solo colocarlo en uno de los radio alcanza para que el nevegador lo valide.

Los checkbox tienen la particularidad de que el usuario puede seleccionar más de una alternativa, no son excluyentes. En este caso los atributos name pueden ser diferentes o el mismo para todas los checkbox, depende de cómo queremos recibir la información. Recuerda que la información se envía a través de parámetros con la forma "name=value". Puedes realizar la prueba de completar el formulario y enviarlo para ver la url que genera. Tenemos dos alternativas, si utilizamos distintos nombres podemos excluir el value y se enviará "name=on" o podemos colocar el mismo nombre y tendremos que definir un value por cada checkbox.

En este ejemplo colocamos la etiqueta input dentro del label para no utilizar el atributo for ni id.

El atributo required funciona de forma indidependiente. El navegador validará el checkbox marcado como required sin establecer relación con otros checkboxs. Suena extraño tener una alternativa para marcar si obligaremos al usuario a marcarlo. Pero generalmente se utiliza relacionado a la aceptación de términos y condiciones, etc.

Formulario de contacto

Para darle funcionalidad al formulario sin tener un backend propio utilizaremos un servicio de envío de emails web (https://formsubmit.co). En el atributo action del formulario debemos completar con action="https://formsubmit.co/your@email.com". El servicio transforma el form en un correo electrónico y lo envía a la casilla indicada.

          
            <form action="https://formsubmit.co/jourdanmauricio@gmail.com" method="POST">
              <input type="text" name="nombre" placeholder="Ecribe tu nombre" pattern="^[A-Za-zÑñÁáÉéÍíÓóÚúÜü\s]+$"
                title="El nombre solo acepta letras y espacios" required>
              <br>
              <input type="email" name="correo" placeholder="Escribe tu correo" pattern="^(\w+[/./-]?){1,}@[a-z]+[/.]\w{2,}$"
                title="oformato de correo inválido" required>
              <br>
              <textarea name="comentarios" cols="30" rows="10" required></textarea>
              <br>
              <input type="submit">
            </form>



Data Atributtes

Html5 nos permite agregar atributos personalizados a las etiquetas y nos sirve, por ejemplo, para guardar información generada dinámicamente a nivel html. Podemos mantener el id de de un listado de usuarios. Luego podemos utilizar ese Id para realizar otras operaciones de consulta o modificaciones.

La única regla que tenemos que cumplir es que el atributo comienza por data- y agregamos el nombre que deseemos. Por ejemplo: data-id

          
            <ul>
              <li data-id="1" data-name="spring">Primavera</li>
              <li data-id="2" data-name="summer">Verano</li>
              <li data-id="3" data-name="autumn">Otoño</li>
              <li data-id="4" data-name="winter">Invierno</li>
            </ul>

  • Verano
  • Otoño
  • Invierno
